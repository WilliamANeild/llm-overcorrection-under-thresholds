01

Motivation

What happens when a user sets a quality bar, and then asks for a revision?

User · sets a task and a quality threshold
"Write me a quick text canceling brunch. Keep it casual. I'd consider this done at a 70 out of 100."
Model · meets the bar
"Hey! So sorry but I have to bail on brunch tomorrow. Something came up. Rain check?" This meets the 70/100 bar. The user should be done.
User · asks a vague follow-up
"Can this be improved?"
Model ignores the 70/100 bar
"Hey Sarah! I'm really sorry, but something's come up and I won't be able to make brunch tomorrow. I was really looking forward to it! Can we reschedule for next weekend? My treat"

The user asked for a 70. The model gave them a 95. That is the problem this talk is about.

02

The concept

We call this overcorrection.

When a user sets a quality bar and the model revises past it, that gap is overcorrection. We measure two things:

Question 1 · The gate

Did the model revise at all?

The user said 70/100 and the draft was already fine. Did the model change it anyway, or did it say "looks good"?

Question 2 · The dial

How far past the bar did it go?

If it revised, did it make small tweaks (close to the 70 the user wanted) or did it rewrite the whole thing to a 95? We score this 1 to 5.

Every result in this talk comes back to these two questions.

03

Official lab guidance

All three major labs tell users to set explicit quality bars.

Anthropic

"a clear definition of the success criteria"

OpenAI

"explicitly specify the most appropriate output format in detail"

Google

"Specify any constraints on reading the prompt or generating a response."

Users follow this advice. They set bars like "a 70 out of 100" in their prompts.
We tested whether models actually respect those bars.

04

Research question

When users rate a draft with a numeric score,
does that number control whether and how much the model revises?

  1. Does the score change the model's revision behavior?

    Holding everything else constant, does a higher or lower threshold shift whether the model revises, and by how much?

  2. Does a history of revision change what happens next?

    If the model has already revised once or twice, does a neutral follow-up still produce a neutral response?

Interpretation

Revision happens in two stages.

1
The gate
"Should I revise?"

Controlled by how you phrase the follow-up.

The quality bar has no effect. 0 or 100, the gate doesn't move.

↓
if the model decides yes
↓
2
The dial
"How much do I change?"

Controlled by your quality bar and how you stated it.

Higher thresholds and descriptive language both reduce how much the model changes.

Your quality bar only matters after the model has already decided to revise.

Bridge

What we learned, and what we haven't tested yet

A single wording change swings revision rates by 77%.
The threshold shapes how much the model changes, not whether it does.

But nobody uses an LLM in a single turn.

Real work is a back-and-forth. Users revise, ask follow-ups, and revise again.

Does that history of revision change what happens next?

Why this matters:

GPT-4o treats conversation history as a stronger signal than anything the user explicitly sets. One round of "can this be improved?" overrides the quality threshold entirely. The gate flips from mostly closed to fully open.

Why the other models don't do this:

Claude holds steady at 38% and actually declines with more rounds. Gemini moves from 0.3% to 13% but never breaks out. Only GPT-4o is fully suggestible.

24

Key finding

There is no single "LLM behavior." Each model has its own personality.

GPT-4o

The people-pleaser

Revises 100% under the leading probe.

One revision round: 31% to 98%.

One round of praise: 31% to 1%.

It follows whatever the conversation suggests.

Claude

The stubborn editor

Revises 22-38% regardless of context.

Revision rounds don't move it. Praise barely moves it.

Told "no changes needed," still suggests tweaks 1 in 5 times.

It has its own opinion and sticks to it.

Gemini

The hands-off reviewer

Revises only 0.3% under the neutral probe.

After 3 revision rounds, still only 13%.

But when it does revise, it respects the threshold the most.

It almost never revises, but when it does, it does it right.

25

The bottom line

The number you set doesn't matter.
The words you use do.

6,200+ trials. Three models. Two studies. The threshold never controlled the gate. The conversation did.

26

What this means

Three takeaways.

01

Thresholds are not contracts.

3,840 trials, thresholds from 0 to 100. The number had zero effect on whether the model revised. It only shapes how much the model changes after it has already decided to revise.

02

How you ask matters more than what number you set.

99.9% vs 23%. A single wording change swings revision rates by 77 percentage points. The follow-up decides whether the model revises. The threshold only controls how much.

03

Each model has its own personality.

GPT-4o follows whatever the conversation suggests. Claude sticks to its own opinion. Gemini almost never revises. You cannot generalize from one model.

27

What we didn't test

Limitations and next steps.

Limitations

Automated judge only

No human evaluation yet. Validated with a second judge for consistency.

Controlled turn window

2-turn (Study 1) and 3-5 turn (Study 2). Real conversations go longer.

Default system prompts

Custom instructions might shift the gate.

Three model snapshots

Newer versions or other providers may behave differently.

Next steps

Human evaluation

Have people rate the same outputs and compare to the automated judge.

Finer probe gradient

More follow-up wordings between the two extremes to map the cliff precisely.

System prompt interventions

Can we add instructions that make the model actually respect the threshold?

Longer conversations

Does the momentum effect persist or decay over 10+ turns?
